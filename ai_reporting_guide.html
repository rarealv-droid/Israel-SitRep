Building live intelligence reports with AI

Part one

Choosing the right tool for the job

Each AI platform has a distinct strength. Use them together rather than picking one — they complement each other at different stages of the reporting workflow.

Tool Where to find it Strengths Limitations Best used for
🧠 Claude
claude.ai
Writes, formats and designs entire reports in one go — structured sections, colour-coded tables, interactive maps and charts
Synthesises multiple sources into coherent, publication-ready analysis
Produces clean HTML that renders directly in a browser, no extra tools needed
Not the fastest at surfacing breaking news — thorough but not instant
Works best when you bring the facts; it builds the structure around them
 Building and updating the full report once you have gathered your data
𝕏 Grok
x.com/i/grok
Real-time access to X (Twitter) — ideal for eyewitness accounts, government announcements and breaking clips within minutes of an event
Fast at pulling raw intelligence across multiple live sources simultaneously
Minimal formatting — output is usually bullet points, not report-ready prose
Can surface unverified or low-credibility posts alongside verified ones; always cross-check before using
 First 30-minute intelligence gathering when speed matters most
🟢 ChatGPT
chatgpt.com
Strong at identifying reliable, named sources — government statements, wire agencies, established outlets
Good at factual verification and spotting contradictions between sources
Plain formatting — requires significant editing to become report-ready
Slower than Grok for truly breaking news; better suited to confirmed and verified developments
 Source-checking and verification before finalising any report
Part two

How to prompt Claude for a full report

You don't need to write code or know design. Think of prompting like briefing a very capable journalist-designer. The more context you give, the better the output. These four steps are all you need.

  1. 1
    Set the brief — tell Claude what you're building

    Start a new Claude conversation and give it the full picture up front: the topic, the audience, what the report should contain, and what it should look like. You're not writing code — you're describing an output the way you'd brief a colleague.

    Example opening prompt → paste into Claude
    I want you to build a daily situation report on [your topic — e.g. "Iranian attacks on Israel"] as a single HTML file that opens in a browser. The report should include: - A header with the title, date, and "OPEN SOURCE ONLY" - A 48-hour executive summary with 4 key categories - Metric cards showing cumulative totals - Key developments section (4 cards in a 2x2 grid) - A normalisation indicators table - Two charts: daily attack waves and interception rates - An interactive map where I can click markers for detail - Collapsible impact tables (direct hits + debris/indirect) - A sources and casualty footnote Use a clean, professional colour scheme — white background, dark text, red for urgent/fatal, amber for significant, green for positive developments. No dark mode. I'll give you the latest data to populate it.
    💡 Tip: You only need to write this opening prompt once. For every future update you just say "regenerate with the latest data" and paste in new information — Claude remembers the format from earlier in the conversation.
  2. 2
    Feed it the data — paste in your gathered intelligence

    Now paste in everything you've gathered — from Grok, from ChatGPT source checks, from wire services, government statements, or your own notes. You don't need to clean it up perfectly. Claude will extract what's relevant and discard the noise.

    Example data prompt → follows your Step 1 brief
    Here is today's raw intelligence. Please build the full report: CASUALTIES: - 3 killed in northern city (Hezbollah rocket, confirmed MDA) - 47 wounded in missile strike on apartment block - Running total: 22 killed, 4,800+ wounded ATTACKS (past 24hrs): - Iran: 13 salvos, one unintercepted hit in Tel Aviv - Hezbollah: dozens of rockets, northern Israel - IDF struck explosives facility near Isfahan DIPLOMACY: - US delivered 15-point plan to Iran via back channels - Pakistan offered to host talks; Trump reshared post - Trump: "They've agreed to never have a nuclear weapon" - Iran publicly denying talks AVIATION: - Lufthansa extended: Dubai/TLV to 31 May, others to Oct 24 - Iraq airspace closed through Wednesday Sources: Times of Israel liveblog 24 Mar, Al Jazeera Day 25, CBS News live, NPR, ITV News, JNS, CNN Day 25 live
    💡 Tip: The more organised your notes, the faster Claude works — but even messy pasted text works. Category labels like CASUALTIES: or DIPLOMACY: help it slot information into the right section automatically.
  3. 3
    Refine — iterate until it looks right

    Claude will produce a full HTML file. Download it and open it in your browser to check it. Then come back and tweak anything that's off. You can be very casual in how you describe changes — you don't need technical language.

    Example refinement prompts — plain English works fine
    // Ask for specific changes in plain language: "The Tel Aviv missile strike should be highlighted in red in the map and tables — it was an interception failure." "Add a note at the top of the exec summary that says attacks are continuing alongside diplomatic talks." "The normalisation indicators section needs a new row for 'Lebanon expels Iran ambassador' with an amber badge." "Can you make the header say Day 26 instead of Day 25?" // For a 12-hour update to an existing report: "Redo this report as a 12-hour update. The key new developments are: [paste new data]. Keep all the existing sections but update the exec summary to reflect the past 12 hours only."
    💡 Tip: Don't try to fix everything in one prompt. Small, specific requests give cleaner results than "fix everything that's wrong." Treat it like a back-and-forth with an editor.
  4. 4
    Download — save the HTML file to your computer

    When you're happy with the report, Claude will give you a download link. Click it to save the .html file. This single file contains everything — the charts, the map, the tables — all self-contained. You can open it in any browser, email it, or upload it to GitHub to share a live link.

    💡 Tip: Name your files consistently — e.g. sitrep_25mar2026.html — so they stay organised and colleagues can see the history at a glance.
    📅 For daily updates: You don't need to start a new conversation each day. Just stay in the same Claude chat, say "regenerate with updates for the past 24 hours", paste in your new data, and Claude will produce a fresh version preserving all the formatting and structure from before.
Part three

Publishing to a live URL with GitHub

GitHub Pages turns your HTML file into a live webpage anyone can open in a browser — no server, no IT ticket, no cost. Setup takes about 10 minutes and you only do it once.

  1. 1
    Create a free GitHub account
    Go to github.com and sign up. Use your work email. Once registered, you're ready — no software to install, everything happens in the browser.
  2. 2
    Create a new repository (think of it as a project folder)
    Click the green New button on your GitHub homepage. Give your repository a name — e.g. Iran-SitRep. Set it to Public (required for free GitHub Pages). Tick "Add a README file" then click Create repository.
    📁 Your repository is now live at github.com/your-username/Iran-SitRep
  3. 3
    Upload your HTML file and name it index.html
    Inside your repository, click Add file → Upload files. Drag in your report HTML. Important: rename it to index.html before uploading — this tells GitHub Pages to treat it as the homepage. Then click Commit changes.
    💡 Tip: Every time you upload a new index.html it automatically overwrites the previous version and your live link updates within about 60 seconds. No extra steps needed for daily updates — just upload and you're done.
  4. 4
    Turn on GitHub Pages
    In your repository, click Settings → scroll down to Pages in the left sidebar → under Source, select Deploy from a branch → choose main branch and / (root) folder → click Save.
    🌐 After ~60 seconds your report is live at your-username.github.io/Iran-SitRep
  5. 5
    Share the link — that's it
    Copy the URL from the Pages settings page and share it with anyone. The link is permanent, loads instantly in any browser, and the interactive features — maps, charts, collapsible tables — all work without any app or plugin. To update the report: generate a new HTML from Claude, rename it index.html, upload it to GitHub, and the live link refreshes automatically.
    🔁 Daily update routine: Generate updated HTML in Claude → download it → rename to index.html → drag into GitHub repository → click Commit. Takes under two minutes once you have the workflow running.
Good luck — the reports look great. 🗺️
